How it works

From observation to automation
in five steps

01

Observe

UserFlow tracks clicks, focus changes, and safe keyboard metadata (shortcuts, Enter, navigation) inside your browser tab only. No raw text. No OS hooks. No clipboard access.

02

Detect

An AI brain analyzes interaction sequences, scores patterns for repetition and confidence, and surfaces automation candidates above the threshold.

03

Review

You see the complete skill before anything activates: trigger, steps, permissions, policies, data used, and estimated time saved. Edit any field before approving.

04

Approve

Check a consent checkbox and click Approve. Nothing activates until you do. The skill enters dry-run-first mode — every trigger shows a preview before acting.

05

Automate

After accepted dry-runs, the skill goes live. Every run is logged. Every action is auditable. You can pause or delete any skill at any time.

Privacy architecture

Built different — by design

🔒

No raw text capture

Keyboard events are reduced to safe metadata only: shortcuts, Enter, navigation keys. Typed content is never stored or transmitted anywhere.

📍

Page-scoped only

Observation is limited to the browser tab UserFlow is active in. No OS-level hooks, no clipboard monitoring, no background activity of any kind.

✅

Explicit consent required

No skill activates without a review dialog, a consent checkbox, and an Approve click. Observers are disabled by default on every page load.

📋

Full audit trail

Every draft, approval, dry-run, activation, and deletion is logged with timestamps, account IDs, and subscription context for compliance.

🛡

Policy engine

Sensitive data patterns, destructive actions, outbound sends, and admin scopes are automatically flagged or blocked before any skill can activate.

🏢

Enterprise deployment

Private deployment, data residency controls, SSO + SCIM provisioning — so your IT and security teams stay in full control of the deployment.
